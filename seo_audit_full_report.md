## 1. EXECUTIVE SUMMARY

kupikod.com — игровой маркетплейс и экосистема для геймеров (ключи, подарочные карты, донат, подписки, медиа-контент). Сайт имеет значительный органический потенциал: по данным xtool.ru, Яндекс видит более **289 000 страниц**, Google — около **200 000 страниц**. Яндекс фиксирует **400 000+ запросов в ТОП-50**. Однако сайт системно теряет трафик из-за нескольких критических технических ошибок.

**Главный вывод:** robots.txt заблокировал **3 096 страниц** из ключевых разделов (/media/, /shop/, /gold/). Это, в совокупности с 96 страницами 500-ошибок, 110 страницами 404, дублями со слешами и массовыми проблемами с длиной title/description — является основной причиной недобора позиций и трафика.

## 2. ГЛАВНЫЕ ПРОБЛЕМЫ САЙТА

### 🔴 КРИТИЧЕСКИЕ (P0)

**1. robots.txt блокирует главные разделы сайта**  
Заблокировано Disallow: /media/ (2 203 страниц контента), /shop/ (505 страниц), /gold/ (338 страниц). Это означает, что тысячи страниц с коммерческим и информационным контентом не обходятся поисковиками (или обходятся значительно реже), не передают внутренний вес, не индексируются нормально.

**2. 96 страниц /media/ возвращают 500 Server Error**  
Статьи блога / медиа-раздела падают с серверной ошибкой. Google и Яндекс выбрасывают такие страницы из индекса. Потеря позиций по информационным запросам.

**3. 110 страниц возвращают 404**  
Среди них — важные коммерческие игровые страницы, которые были удалены: /fortnite, /minecraft, /pubg, /roblox/cards, /valorant, /brawl-stars — все это высокочастотные запросы с огромным коммерческим потенциалом.

**4. Canonical конфликты (trailing slash)**  
6 ключевых URL имеют несоответствие с canonical: /console → /console/, /gold → /gold/, /shop/ → /shop, /terms/ → /terms. Каждая пара — это фактически дубль страницы.

### 🟠 ВЫСОКИЙ ПРИОРИТЕТ (P1)

**5. 58% страниц имеют Title > 60 символов**  
3 854 страницы с обрезанным title в выдаче. Ключевые слова теряются, CTR падает.

**6. 47% страниц имеют Description > 160 символов**  
3 122 страницы — сниппеты обрезаются, Google генерирует свой сниппет.

**7. Страницы без H1: 15 страниц**  
Включая ключевые: /media/novosti, /media/stati, /media/promokody, /promocode — критично.

**8. Главная страница — 166 слов**  
Катастрофически мало для главной страницы коммерческого проекта. Нет описания, нет FAQ, нет USP.

**9. /promocode — нет H1**  
Коммерческая страница промокодов без заголовка.

### 🟡 СРЕДНИЙ ПРИОРИТЕТ (P2)

**10. Скорость сервера — avg 1.095s, 866 страниц > 1s, max 29.2s**  
Core Web Vitals страдают. Google напрямую учитывает скорость в ранжировании.

**11. Глубина страниц до depth 55**  
Часть страниц недоступна для поисковиков из-за слишком большой вложенности.

**12. 18 страниц с множественными H1, 12 групп дублей H1**

**13. Числа в H1: "Игры 7991", "Игровая валюта 102", "Подписки 21"**  
Плохая практика. H1 должен содержать ключевые слова, а не счётчик товаров.
